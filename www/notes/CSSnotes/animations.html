CSS Keyframe animations

Config

Each element you are intending to element gets what I like to think of as a kind of config file in which you define the overall/main arc animation settings:

.element {
  animation-name: whereTheKeyframesAre; //the animation itself is defined elsewhere so we can reuse specific movments, like a bounce!
  animation-duration: 4s;
  animation-delay: 1s;
  animation-iteration-count: infinite; //or set a number
  animation-timing-function: linear;  
  animation-direction: alternate;
}

the animation-timing-function is where we can use some present animation curves rather than work it out ourselves

  • linear: straight - even speed from start to finish, robotic, hard start and end.
  • ease: simple curve - slow start slow finish, more organic.
  • ease-in: slow start
  • ease-out: slow end
  • ease-in-out: similar to ease but not as dramatic
  • cubic-bezier(n,n,n,n): set our own values, here's a tool to set the bezier curves visually
The animation-direction also has a few presets for us to play with:
  • normal It will play from start to finish, if repeated it will jump to the start for every animation
  • reverse
  • alternate
  • alternate-reverse begins with a reverse animation
Shorthand: 
animation: name duration delay iteration-count timing-function direction;


@keyframes

This is where the work happens. As the length/time of the animation is already defined in the 'config' above we set up the stages of the animation in %, a little foreign when coming from actual animation but great for reusability!

@keyframes whereTheKeyframesAre {
    0%   {top: 0px;     left: 0px;    background: red;} //it's good to define 0 and 100 to help with cross browser stuff
    25%  {top: 0px;     left: 100px;  background: blue;}
    50%  {top: 100px;   left: 100px;  background: yellow;}
    75%  {top: 100px;   left: 0px;    background: green;}
    100% {top: 0px;     left: 0px;    background: red;}
}

Multiple animations

We can add a second animation in by seporating the two with a comma. They will play simultaniously as shown in the example below:

  width: 200px;
  height: 200px;
  background-color: black;
  -webkit-animation: 
      corners 2s .5s infinite ease alternate, 
      colours 2s .5s infinite ease alternate;

A note for the animation-timing-function

You may have realised that the timing function is applied between the movment between every defined keyframe which is why there appears to be 4 clearly defined transitions in the previous example. If we take the timing defenition out of the config and apply it to the keyframes we can apply timing accros the animation as a whole.

@keyframes corners {
  0%  { border-radius:0 0 0 0; animation-timing-function: ease-in; }
  25%  { border-radius:50% 0 0 0; animation-timing-function: linear; }
  50%  { border-radius:50% 50% 0 0; animation-timing-function: linear; }
  75%  { border-radius:50% 50% 50% 0; animation-timing-function: ease-out; }
  100% { border-radius:50%; animation-timing-function: ease-in; }
}
